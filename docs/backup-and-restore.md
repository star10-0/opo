# Backup and Restore Guide

## ما الذي يجب نسخه احتياطيًا
- قاعدة بيانات MongoDB كاملة (كل الـ collections).
- ملف `.env` الخاص ببيئة الإنتاج (يحفظ خارج المستودع وبشكل آمن).
- أي ملفات تشغيل/تهيئة نشر خارجية مرتبطة بالخدمة.

## إنشاء نسخة احتياطية (MongoDB)
```bash
mongodump --uri="$MONGO_URI" --out ./backups/$(date +%F-%H%M%S)
```

## استعادة نسخة احتياطية
> تحذير: الاستعادة قد تستبدل بيانات موجودة. نفّذها في نافذة صيانة.

```bash
mongorestore --uri="$MONGO_URI" --drop ./backups/<backup-folder>
```

## التحقق بعد الاستعادة
1. شغّل backend وتحقق من `GET /api/health`.
2. تحقق من عدد السجلات الأساسية (stations, operational-days, worker-closings).
3. نفّذ تقرير يومي من الواجهة للتأكد من سلامة العلاقات.

## حد أدنى لسياسة النسخ
- نسخة يومية تلقائية + نسخة أسبوعية محفوظة فترة أطول.
- حفظ نسخة خارج الخادم الأساسي (offsite).
- اختبار الاستعادة دوريًا على بيئة staging.
