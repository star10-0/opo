# النسخ الاحتياطي والاستعادة (MongoDB)

## ما الذي يجب نسخه احتياطيًا
- قاعدة بيانات MongoDB كاملة الخاصة بالنظام.
- ملف البيئة الخاص بالخلفية `.env` (بدون مشاركته علنًا).
- أي ملفات تشغيل/نشر تحتوي إعدادات الإنتاج.

## أوامر النسخ الاحتياطي
> عدّل القيم بما يناسب بيئتك.

```bash
mongodump \
  --uri="$MONGO_URI" \
  --out="./backups/mongo-$(date +%F-%H%M)"
```

لضغط النسخة الاحتياطية:

```bash
tar -czf "./backups/mongo-$(date +%F-%H%M).tar.gz" ./backups/mongo-*
```

## أوامر الاستعادة
> يفضّل الاستعادة في بيئة اختبار أولًا.

```bash
mongorestore \
  --uri="$MONGO_URI" \
  --drop \
  ./backups/mongo-YYYY-MM-DD-HHMM
```

## التحقق بعد الاستعادة
1. تشغيل backend والتأكد من `/api/health`.
2. تسجيل الدخول والتحقق من تحميل المحطات.
3. فتح تقارير يومية والتأكد من وجود البيانات.

## ملاحظات تشغيلية
- اجعل النسخ الاحتياطي دوريًا (يومي/أسبوعي حسب حجم البيانات).
- احتفظ بنسخ متعددة (على الأقل آخر 7 نسخ).
- خزّن نسخة خارج الخادم الرئيسي.
- لا تحفظ أسرار التشغيل في المستودع.
