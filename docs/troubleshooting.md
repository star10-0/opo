# دليل المشاكل الشائعة والحلول

## 1) لا تظهر البيانات في الصفحات
**الأعراض**
- الصفحة تفتح لكن القوائم فارغة أو تظهر رسالة خطأ متقطع.

**التحقق**
1. تحقق أن `stationId` مختار من الواجهة.
2. تحقق من نجاح `/api/health`.
3. راجع Console/Network في المتصفح لرموز 401/403/500.

**الحل**
- اختر محطة صحيحة.
- تأكد من تسجيل الدخول بصلاحية مناسبة.
- تحقق من `VITE_API_BASE_URL` وأنه يشير للـ backend الصحيح.

## 2) لا يمكن اختيار محطة
**الأسباب المحتملة**
- حساب المستخدم غير مرتبط بأي محطة.
- API المحطات فشل في التحميل.

**الحل**
- اطلب من المدير إضافة المحطة إلى `stationAccess` للمستخدم.
- أعد تسجيل الدخول بعد تحديث الصلاحيات.

## 3) Backend لا يعمل
**الأعراض**
- فشل `npm run dev` في مجلد backend.
- endpoint الصحة لا يستجيب.

**التحقق**
- تحقق من `MONGO_URI`.
- تحقق من وجود `JWT_SECRET`.
- تحقق من منافذ التشغيل وعدم تعارضها.

**الحل**
```bash
cd backend
npm install
npm run dev
npm run healthcheck
```
- إن استمر الخطأ، افحص log الإقلاع وحدد أول exception.

## 4) Frontend لا يظهر التحديثات
**الأعراض**
- حفظ العملية ينجح لكن الشاشة لا تتحدث فورًا.

**التحقق/الحل**
1. قم بتحديث الصفحة يدويًا للتأكد أن البيانات حُفظت.
2. تأكد أن الطلبات بعد الحفظ تُعيد تحميل البيانات (refetch).
3. افحص Network للتأكد من عدم وجود caching غير مقصود.

## 5) لا تظهر التقارير
**الأسباب المحتملة**
- لا توجد بيانات كافية للفترة المحددة.
- الصلاحيات لا تسمح برؤية التقارير.
- خطأ في معلمات التاريخ أو المحطة.

**الحل**
- اختر نطاق تاريخ صحيح.
- تأكد من وجود يوم تشغيلي/قراءات/إغلاقات فعلية.
- تحقق من صلاحية `view_reports`.

## 6) لا يمكن الاعتماد أو الحفظ
**الأسباب المحتملة**
- نقص صلاحيات الدور.
- سجل مؤرشف يحتاج Approval Request.
- حقول إلزامية ناقصة.

**الحل**
- تحقق من الدور الحالي.
- في السجلات المؤرشفة: أنشئ طلب موافقة بدل التعديل المباشر.
- أكمل الحقول المطلوبة ثم أعد المحاولة.

## 7) مشاكل اللغة أو الصلاحيات
**اللغة**
- تأكد من اختيار العربية من محدد اللغة.
- إن تغيرت اللغة بشكل غير متوقع، أعد ضبطها من الواجهة ثم أعد فتح الصفحة.

**الصلاحيات**
- قد لا تظهر التبويبات بسبب صلاحيات الدور.
- راجع مصفوفة الأدوار في `docs/roles-and-permissions.md`.
