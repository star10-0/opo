# دليل التشغيل الداخلي

> مخصص للمشغل التقني/مدير النظام لتشغيل المشروع محليًا والتحقق من الجاهزية.

## 1) المتطلبات الأساسية
- Node.js (إصدار حديث متوافق مع المشروع)
- npm
- MongoDB شغال محليًا أو عبر URI خارجي

## 2) أماكن ملفات البيئة
- الجذر: `.env.example`
- Backend: يستخدم متغيرات البيئة عند التشغيل من مجلد `backend`
- Frontend: يعتمد على متغيرات `VITE_*` (خصوصًا `VITE_API_BASE_URL`)

### إعداد أولي سريع
1. انسخ ملف البيئة:
   ```bash
   cp .env.example .env
   ```
2. حدّث القيم الأساسية في `.env`:
   - `MONGO_URI`
   - `JWT_SECRET`
   - `FRONTEND_URL`
   - `VITE_API_BASE_URL`

## 3) تشغيل الـ Backend
```bash
cd backend
npm install
npm run dev
```

### التحقق من عمل الـ Backend
- تحقق من الصحة:
  ```bash
  npm run healthcheck
  ```
- أو افتح endpoint الصحة مباشرة: `/api/health`.

## 4) تشغيل الـ Frontend
```bash
cd fuel-frontend-v2
npm install
npm run dev
```

- افتح الرابط الذي يعرضه Vite (عادة `http://localhost:5173`).

## 5) التحقق أن النظام يعمل end-to-end
1. تسجيل الدخول من الواجهة.
2. التأكد من تحميل لوحة التحكم بدون أخطاء حرجة.
3. اختيار محطة.
4. تنفيذ عملية أساسية (مثل عرض اليوم التشغيلي أو التقارير).

## 6) تحديث الإعدادات الأساسية
عند الحاجة لتعديل سلوك النظام:
- إعدادات API/الربط: عدّل `VITE_API_BASE_URL` ثم أعد تشغيل الواجهة.
- إعدادات الأمان/الخلفية: عدّل `ENFORCE_AUTH`, `JWT_SECRET`, `NODE_ENV` ثم أعد تشغيل backend.
- إعدادات CORS: تأكد من `FRONTEND_URL` الصحيح.

## 7) التعامل مع البداية الأولى للنظام
عند قاعدة بيانات جديدة فارغة:
1. سجّل دخولك بحساب مدير.
2. من لوحة التحكم أنشئ "أول محطة" باستخدام بطاقة الإنشاء.
3. (اختياري) فعّل إنشاء بيانات بدء مبدئية (خزانات/مضخات).
4. اربط المستخدمين بالمحطة عبر صلاحيات الوصول للمحطات.
5. ابدأ يومًا تشغيليًا تجريبيًا للتحقق من دورة العمل.

## 8) Runbook قصير للتسليم
- [ ] backend يعمل و healthcheck ناجح.
- [ ] frontend يعمل ويرتبط بالـ API الصحيح.
- [ ] يوجد حساب مدير فعّال.
- [ ] تم إنشاء محطة واحدة على الأقل.
- [ ] تم اختبار التقارير الأساسية.
- [ ] تم حفظ نسخة من الإعدادات المعتمدة للتشغيل.
